## 🚀 How to Deploy on GitHub Pages

### Step 1 — Create a GitHub Repository
1. Go to [github.com](https://github.com) and sign in
2. Click the **+** icon → **New repository**
3. Name it `acr-engineering` (or any name you prefer)
4. Set visibility to **Public**
5. Click **Create repository**

### Step 2 — Upload the Files
1. On your new repository page, click **Add file** → **Upload files**
2. Drag and drop `index.html` and `README.md` into the upload area
3. Scroll down, add a commit message like `Initial website upload`
4. Click **Commit changes**

### Step 3 — Enable GitHub Pages
1. Go to your repository → **Settings** tab
2. Scroll down to **Pages** (in the left sidebar)
3. Under **Source**, select **Deploy from a branch**
4. Set Branch to **main** and folder to **/ (root)**
5. Click **Save**

### Step 4 — Access Your Live Site
- Wait 1–2 minutes for GitHub to build the site
- Your site will be live at: `https://[your-username].github.io/acr-engineering/`
- GitHub will show a green banner with the live URL in the Pages settings

To update the site after changes:
1. Go to your repository on GitHub

4. Commit the changes — GitHub Pages will automatically redeploy within 1–2 minutes
